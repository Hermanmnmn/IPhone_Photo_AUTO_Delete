# iPhone 照片自動清理系統安裝與設定指南 (README.md)

歡迎使用 **iPhone 照片自動清理系統**（「閱後即焚」照片清理系統）。本系統完全基於 iOS 內建的「捷徑（Shortcuts）」與「個人自動化」功能實現。在您開啟自動刪除模式後，所有新拍攝的照片、影片與螢幕截圖，都會在 **7 天後** 自動移入相簿中的「最近刪除」檔案夾，讓您的相簿保持乾淨。

本系統不需要任何第三方應用程式或開發者帳號，所有狀態與待刪除清單均安全地儲存在您的 iCloud 雲碟中。並且**支援自動初始化**，無須手動建立任何檔案與資料夾，匯入後即可直接使用！

---

## 🚀 快速安裝（推薦）：直接匯入已簽署的 .shortcut 檔案

我們已為您生成好 3 個最新的受信任簽署 `.shortcut` 檔案，您只需傳送到 iPhone 即可直接使用。

### 匯入步驟：

1. 在 Mac 桌面上找到 **`iPhone自動刪除捷徑`** 資料夾中的 3 個 `.shortcut` 檔案。
2. 透過 **AirDrop**（推薦）選取所有檔案傳送到你的 iPhone。
3. 在 iPhone 上收到後選擇「**儲存到檔案**」，並存放在「檔案」App 中。
4. 打開 iPhone 的 **「檔案」** App，依序點擊這 3 個檔案，在彈出的系統視窗中點擊 **「加入捷徑」**：
   - `1_切換自動刪除模式.shortcut`
   - `2_自動標記新照片.shortcut`
   - `3_執行自動刪除清理.shortcut`
5. 匯入完成後，直接前往 **第 4 節** 設定自動化、**第 5 節** 設定控制中心、**第 6 節** 設定隱私權限。

## 1. 系統架構與資料流 (System Architecture & Data Flow)

本系統由 **3 個 iOS 捷徑** 與 **2 個個人自動化觸發器** 組成，它們藉由 iCloud 雲碟進行資料的持久化讀寫與溝通。以下是系統的整體架構與資料流向圖：

```
                      +-------------------+
                      |   iOS 18 控制中心  |
                      +---+---------------+
                          | (點擊觸發/顯示狀態)
                          v
                    +-----+---------------+
                    | 1_切換自動刪除模式   |
                    |  (Toggle Shortcut)  |
                    +----------+----------+
                               | (讀取/自動初始化)
                               v
    +---------------------------+----------------------------+
    |                     iCloud Drive                       |
    |              Shortcuts/AutoDeletePhotos/               |
    |                                                        |
    |  +------------------+          +--------------------+  |
    |  |    status.txt    |          |  last_tag_time.txt |  |
    |  | ("開啟" / "關閉") |          |   (上次標記時間)   |  |
    |  +--------+---------+          +--------+-----------+  |
    |           ^                             ^              |
    |           | (讀取狀態是否開啟)            | (讀取/更新)   |
    |           +--------------+--------------+              |
    |                          |                             |
    |                          v                             |
    |              +-----------+----------+                  |
    |              |   2_自動標記新照片   |                  |
    |              |  (Tagging Shortcut)  |                  |
    |              +-----+----------+-----+                  |
    |                    |          |                        |
    |               (尋找新相片)   (讀取/寫入)                |
    |                    |          v                        |
    |                    |   +------+-----------+            |
    |                    |   |pending_list.json |<-----------+
    |                    |   | (待刪除照片清單)  |
    |                    |   +------+-----------+
    |                    v          ^
    |            +-------+------+   | (讀取清單並寫回)
    |            |  iOS 照片庫  |   |
    |            +-------+------+   |
    |                    ^          |
    |                    | (刪除)   |
    |                    |          |
    |              +-----+----------+-----+
    |              | 3_執行自動刪除清理   |
    |              |  (Cleanup Shortcut)  |
    |              +----------^-----------+
    |                         |
    +-------------------------|------------------------------+
                              | (定期/定時觸發)
                      +-------+-------+
                      | iOS 個人自動化| (例如每週自動執行一次)
                      +---------------+
```

### 系統元件說明：
1. **1_切換自動刪除模式**：使用者點擊時會直接在背景切換「開啟/關閉」狀態，並在螢幕頂部彈出對應的系統通知。完全自動化，免除彈窗與選單確認，非常適合控制中心背景執行。
2. **2_自動標記新照片**：每當您完成拍照或關閉相機時，自動化觸發此捷徑。若模式為 `開啟`，它會獲取所有新照片（晚於 `last_tag_time.txt`），預設以 **7 天 (10080 分鐘)** 進行過期時間計算，並寫入 `pending_list.json` 中。
3. **3_執行自動刪除清理**：定時自動化（例如每週一次）觸發此捷徑。它會比對 `pending_list.json` 的過期時間，並將過期照片移入最近刪除，更新並保留未到期照片。

## 2. iCloud Drive 目錄與檔案格式 (Folder Structure & File Formats)

所有資料皆儲存在 iCloud 雲碟的指定目錄中。

* **持久化根目錄**：`iCloud 雲碟/Shortcuts/AutoDeletePhotos/`

> **注意**：在 iOS 捷徑的「檔案」動作中，預設根目錄即為 `Shortcuts/`，因此在設定捷徑動作的路徑時，請填寫相對於該根目錄的相對路徑，即：`AutoDeletePhotos/`。

### 檔案清單與規格說明

#### 1. `status.txt`
* **用途**：持久化記錄自動刪除模式的啟用狀態。
* **格式**：純文字。
* **容許值**：`開啟` 或 `關閉`（請確保無換行與額外空白）。
* **容錯機制**：若此檔案遺失，捷徑將預設其狀態為 `關閉`。

#### 2. `pending_list.json`
* **用途**：記錄目前排程待刪除的照片清單。
* **格式**：JSON 陣列（Array of Objects）。
* **欄位結構**：
  * `id` (String): 照片的唯一檔名（例如 `IMG_1234.HEIC`）。
  * `captured_at` (String): 照片被拍攝並標記的 ISO 8601 時間戳記（含時區，例如 `2026-07-16T17:35:43+08:00`）。
  * `delete_at` (String): 預定自動刪除該照片的 ISO 8601 時間戳記（含時區，例如 `2026-07-16T18:05:43+08:00`）。
* **JSON 範例**：
  ```json
  [
    {
      "id": "IMG_1024.HEIC",
      "captured_at": "2026-07-16T17:35:43+08:00",
      "delete_at": "2026-07-16T18:05:43+08:00"
    },
    {
      "id": "IMG_1025.PNG",
      "captured_at": "2026-07-16T17:40:00+08:00",
      "delete_at": "2026-07-16T19:40:00+08:00"
    }
  ]
  ```
* **容錯機制**：若讀取不到此檔案，捷徑會自動將其視為空陣列 `[]`，並在下次寫入時建立該檔案。

#### 3. `last_tag_time.txt`
* **用途**：記錄上一次成功標記照片的時間戳記。用於判斷哪些照片是「新拍攝」的，確保**一次拍攝多張照片時全部都會被標記**（而非只標記最新的 1 張）。
* **格式**：純文字（ISO 8601 時間戳記）。
* **範例**：`2026-07-16T17:35:43+08:00`
* **容錯機制**：若此檔案不存在（首次使用），系統會安全地僅標記最新的 1 張照片，並在標記後建立此檔案。

## 3. 每個捷徑的逐步建立教學 (Step-by-step Shortcuts Creation)

> 💡 **提示**：強烈建議使用前面的 **「🚀 快速安裝」** 直接匯入已簽署的 `.shortcut` 檔案。如果您偏好手動在 iPhone 上一步步建立捷徑，以下是各捷徑的動作與核心邏輯清單：

---

### 捷徑一：1_切換自動刪除模式 (Toggle Shortcut)
* **目的**：提供控制中心開關，點擊時直接在背景切換狀態，免去選單確認與交互，極致流暢。

#### 核心動作邏輯：
1. **取得檔案**：`AutoDeletePhotos/status.txt` (如果找不到即為空) 並設定成變數 `StatusFile`。
2. **將檔案轉為文字**：讀取變數 `StatusFile` 的文字 (若不存在，此文字為空) 並設定成變數 `StatusText`。
3. **判斷狀態是否為「開啟」**：比對變數 `StatusText` 是否等於「開啟」：
   - **若是** (目前為開啟狀態)：
     - 寫入 `關閉` 至 `AutoDeletePhotos/status.txt` (覆寫)
     - 發送系統通知：「🔴 自動刪除模式已關閉」
   - **否則** (目前為關閉狀態或首次執行檔案不存在)：
     - 寫入 `開啟` 至 `AutoDeletePhotos/status.txt` (覆寫)
     - 發送系統通知：「🟢 自動刪除模式已開啟 (7天自動刪除)」

---

### 捷徑二：2_自動標記新照片 (Photo Tagging)
* **目的**：相機關閉時執行。若開啟模式，則標記所有晚於 `last_tag_time.txt` 的新照片，並寫入待刪除清單。

#### 核心動作邏輯：
1. **讀取狀態**：讀取 `status.txt` 並設定為變數 `StatusFile`，轉換成 `StatusText`。若 `StatusText` 等於 `開啟` 則繼續。
2. **取得當前時間**：設定 `NowISO`，並將其加 10080 分鐘 (7天) 得到 `DeleteAtISO`。
3. **讀取待刪除清單**：讀取 `pending_list.json` 並設定為變數 `ListFile`，轉換成 `ListText` 後載入現有資料。
4. **讀取上次標記時間**：讀取 `last_tag_time.txt` 並設定為 `LastTagFile`，轉換為 `LastTagText` 後做為判斷依據。
5. **尋找相片**：
   - 若 `LastTagText` 有值：尋找最新 20 張照片。
   - 否則 (首次)：僅尋找最新 1 張照片。
6. **重複檢查照片**：
   - 對於每一張相片，取得其「拍攝日期」。
   - 若 `LastTagText` 有值，且拍攝日期晚於上次標記時間（或是首次執行），則：
     - 取得檔名 (PhotoID)
     - 建立字典：`{"id": PhotoID, "captured_at": NowISO, "delete_at": DeleteAtISO}`
     - 將字典加入變數 `PhotoList` 及暫存變數 `NewTags`
7. **儲存檔案**：
   - 將 `PhotoList` 變數寫入 `pending_list.json` (覆寫)。
   - 將 `NowISO` 寫入 `last_tag_time.txt` (覆寫)。
8. **發送通知**：計算 `NewTags` 數量存入 `NewTagsCount`。若 `NewTagsCount` 大於 `0`，則發送通知。

---

### 捷徑三：3_執行自動刪除清理 (Cleanup Shortcut)
* **目的**：定時每週在背景自動清理已到期的照片。

#### 核心動作邏輯：
1. **讀取清單**：讀取 `pending_list.json` 並設定為變數 `ListFile`，轉換成 `ListText` 後獲取所有字典物件 `AllEntries`。
2. **重複檢查每一個項目**：
   - 取得項目的 `delete_at` 並解析為日期 `DeleteAtDate`。
   - 若當前時間晚於 `DeleteAtDate` (已過期)：
     - 根據項目中的 `id` 尋找照片。
     - 若找到照片，則呼叫「刪除照片」動作移入「最近刪除」相簿。
     - 將標記 `1` 加入 `DeletedItems` 變數中。
   - 否則 (未過期/解析失敗)：
     - 將該項目加入變數 `KeepList`。
3. **儲存檔案**：將 `KeepList` 寫回 `pending_list.json`。
4. **發送通知**：計算 `DeletedItems` 數量存入 `DeletedCount`。若 `DeletedCount` 大於 `0`，則發送通知。

## 4. 自動化設定教學 (Automation Setup)

為了讓系統完全自動在背景運作，我們需要設定 iOS 個人自動化來自動執行「標記」與「清理」捷徑。

### 1. 設定新照片標記觸發器 (Photo Tagging Trigger)
每當您拍完照關閉相機時，此自動化會背景執行「自動標記新照片」。

* **設定步驟**：
  1. 打開「捷徑」App，切換至底部的 **「自動化」** 分頁。
  2. 點擊右上角的 **「+」** (或「建立個人自動化」)。
  3. 選擇 **「App」** 作為觸發條件。
  4. 點擊「App：選擇」，勾選 **「相機」**，然後點擊「完成」。
  5. 勾選 **「已關閉」**，並取消勾選「已開啟」。
  6. 在下方點選並勾選 **「立即執行」** (Run Immediately)，並確認將 **「執行時通知」** 關閉。
  7. 點擊「下一步」。
  8. 搜尋並點選 **「執行捷徑」** (Run Shortcut) 動作，點擊「捷徑」字樣，選擇 **`2_自動標記新照片`**。
  9. 點擊右上角的「完成」儲存。

### 2. 設定每週清理自動化 (Weekly Cleanup Automation)
此自動化將定期（如每週一次）在背景自動清除所有已到期的照片。

* **設定步驟**：
  1. 在「捷徑」App 中的 **「自動化」** 分頁，點擊右上角的 **「+」**。
  2. 選擇 **「特定時間」**。
  3. 將執行時間設定為：`每週`，並選擇一個您通常不使用手機的時間（例如：`星期日 02:00`）。
  4. 在下方點選並勾選 **「立即執行」** (Run Immediately)，並確認將 **「執行時通知」** 關閉。
  5. 點擊「下一步」。
  6. 搜尋並點選 **「執行捷徑」** 動作，點擊「捷徑」字樣，選擇 **`3_執行自動刪除清理`**。
  7. 點擊右上角的「完成」儲存。

## 5. 控制中心設定教學 (Control Center Setup)

iOS 18 允許您將捷徑直接加入下拉控制中心，方便您隨時點擊切換與檢視模式。

```
+------------------------------------+
|  [控制中心編輯畫面]                 |
|                                    |
|          +--------------+          |
|          | 🔄 自動刪除  |          |
|          |  (切換開關)  |          |
|          +--------------+          |
|                                    |
+------------------------------------+
```

* **設定步驟**：
  1. 在 iPhone 螢幕右上角向下滑動，打開 **「控制中心」**。
  2. 在控制中心的空白處長按，或點擊頂部的 **「編輯」** 進入控制中心編輯模式。
  3. 點擊底部的 **「新增控制項目」** (Add a Control)。
  4. 在搜尋框中輸入「捷徑」(Shortcuts)，選擇「捷徑」控制項（按鈕圖示）。
  5. 點擊新加入的捷徑控制項，在彈出的設定選單中選擇 **`1_切換自動刪除模式`** 捷徑。
  6. 點擊螢幕任何空白處完成編輯。
  7. 現在，您只需滑開控制中心，點擊它即可直接在背景切換狀態並彈出通知！

## 6. 隱私權限設定教學 (Privacy Settings)

為了實現「完全自動無感刪除」，我們必須授權清理捷徑「刪除不詢問」的隱私權限，否則每次執行刪除時，iOS 都會彈出確認視窗，破壞自動化體驗。

* **設定步驟**：
  1. 打開「捷徑」App，切換至 **「捷徑」** 分頁。
  2. 尋找 **`3_執行自動刪除清理`** 捷徑，點擊卡片右上角的 **「...」** (三個點) 按鈕進入編輯畫面。
  3. 在編輯畫面最下方，點擊正中間的 **「資訊 (i)」** 圖示 (iOS 18 中位於螢幕正底部的圈圈 i)。
  4. 點選並切換至 **「隱私」** 分頁。
  5. 在隱私權限清單中，找到 **「照片」** 選項：
     - 將權限更改為 **「允許存取」**。
     - 在「刪除照片」或「允許刪除」的進階開關下，將 **「刪除不詢問」** (Delete without asking) 開關 **開啟** (啟用)。
  6. 點擊右上角的「完成」儲存。

## 7. 測試與驗證流程 (Test Walkthrough)

請依照以下步驟驗證您的系統是否設定成功：

### 第一階段：開啟系統並拍照
1. 打開「控制中心」，點擊 **`1_切換自動刪除模式`**。
2. 此時會直接在螢幕頂部彈出系統通知，顯示已開啟或已關閉。如果目前是關閉狀態，點選後會直接切換為「🟢 自動刪除模式已開啟 (7天自動刪除)」。
3. 使用 iPhone 相機拍攝一張測試照片。
4. 完全關閉相機 App（向上滑動關閉）。此時會觸發 `2_自動標記新照片` 自動化。
5. 檢查是否成功收到通知：「新照片已標記，📸 已排程 1 張照片，將在 7 天後刪除」。

### 第二階段：驗證資料檔案是否成功自動初始化
1. 打開 iPhone 內建的 **「檔案」** App。
2. 進入 `iCloud 雲碟 > Shortcuts > AutoDeletePhotos` 目錄。
3. 檢查以下檔案是否正常建立且內容正確：
   - `status.txt` 內容應為：`開啟`
   - `pending_list.json` 內容應包含剛拍下照片的檔名與 7 天後的到期時間。
   - `last_tag_time.txt` 內容應為剛剛拍攝的時間戳記。

### 第三階段：測試未到期保留
1. 在照片還沒到期前，打開「捷徑」App 手動點擊執行 **`3_執行自動刪除清理`** 捷徑。
2. 檢查照片是否依然**完好保留**在相簿中，確認未到期相片不會被誤刪。

### 第四階段：測試到期自動刪除
為避免實際等待 7 天，我們手動將到期時間調小來驗證刪除邏輯：
1. 打開「檔案」App，導航至 `Shortcuts/AutoDeletePhotos/pending_list.json`。
2. 點擊檔案進行編輯：將內容中的 `"delete_at"` 時間戳記修改為**昨天或一小時前**的時間，然後儲存。
3. 打開「捷徑」App，手動點擊執行 **`3_執行自動刪除清理`** 捷徑.
4. 執行完畢後，應收到通知：「自動清理完成，🗑️ 已清理 1 張已到期的過期照片」。
5. 打開手機的 **「照片」** App，您會發現剛才那張測試照片已被移至 **「最近刪除」** 相簿中。驗證成功！

## 8. 常見問題與故障排除 (FAQ & Troubleshooting)

### Q1：自動清理照片時，手機依然跳出「允許捷徑刪除這張照片嗎？」的彈窗？
* **解決方法**：這是因為「刪除不詢問」隱私權限未正確啟用。請參考本指南 **第 6 節「隱私權限設定教學」**，重新進入 `3_執行自動刪除清理` 捷徑的資訊隱私分頁，確保將「照片」的存取權限完全開啟，並啟用「刪除不詢問」功能。

### Q2：拍照關閉相機後，沒有收到「新照片已標記」的通知？
* **解決方法**：
  1. 請檢查您的 `status.txt` 內容是否為 `開啟`。如果模式是關閉的，系統會靜默跳過標記。
  2. 請確保在「個人自動化」中，已將觸發器設定為「立即執行」並關閉「執行時通知」，且確實執行了 `2_自動標記新照片` 捷徑。
  3. 檢查「捷徑」App 是否被允許存取您的相簿權限。可至 iOS「設定」 > 「捷徑」中確認「照片」存取為「所有照片」。

### Q3：如果在倒數時間到期前，我自己把照片手動刪除了，系統會報錯嗎？
* **解決方法**：不會。系統設計了完善 of 異常防錯機制。當清理捷徑在 `pending_list.json` 發現該筆紀錄但去相簿尋找照片時，若發現該照片已不存在（被手動刪除），清理捷徑會靜默忽略該錯誤，繼續清理下一個項目，並在最後將該筆紀錄從待刪除清單 JSON 檔案中正常移除，防止清單無限膨脹。

### Q4：在飛航模式（完全沒有網路）下拍照，系統能正常運作嗎？
* **解決方法**：可以。iOS 捷徑會將資料寫入本地 iCloud 雲碟的緩存中，標記與寫入 JSON 動作在本地即可完成。等裝置重新連接網路後，iCloud 雲碟會自動將檔案同步至雲端，不影響清理捷徑的執行。

### Q5：修改手機的時間或時區，會影響刪除時間的判定嗎？
* **解決方法**：照片的到期時間是使用精確的 ISO 8601 格式儲存（包含時區偏移行，如 `+08:00`）。清理捷徑在執行時，會將其與當前系統時間進行比對。如果您手動更改時區或系統時間，可能會使「Now 晚於或等於 DeleteAt」的判定產生提前或延遲。建議手機維持自動校對時間設定。

### Q6：如何將此系統徹底移除？
* **解決方法**：
  1. 點選執行 `切換自動刪除模式`，直到螢幕頂部彈出通知顯示「🔴 自動刪除模式已關閉」。
  2. 前往「捷徑」App 中的「自動化」分頁，將關閉相機自動化與每週清理自動化向左滑動刪除。
  3. 打開「檔案」App，手動將整個 `iCloud 雲碟/Shortcuts/AutoDeletePhotos/` 資料夾刪除即可。
